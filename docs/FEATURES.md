## 🚀 Phase 1: 基础增强

### 核心功能

#### 1. 问题类型检测（5种）

自动识别用户问题的类型：

| 类型 | 说明 | 示例问题 |
|------|------|---------|
| **描述型** | 是什么 | "什么是PE比率？" |
| **分析型** | 为什么 | "为什么宁德时代涨了？" |
| **建议型** | 怎么办 | "现在适合买吗？" |
| **比较型** | 对比 | "A和B哪个好？" |
| **预测型** | 未来 | "未来走势如何？" |

**技术实现：**
- 关键词匹配
- 句式分析
- 意图推断

#### 2. 复杂度评估（3级）

自动评估问题的复杂程度：

| 级别 | 说明 | 研究时间 |
|------|------|---------|
| **简单** | 事实性问题 | 5-10分钟 |
| **中等** | 需要分析推理 | 10-20分钟 |
| **复杂** | 深度研究 | 20-30分钟 |

**评估维度：**
- 问题类型
- 涉及范围
- 时间跨度
- 实体数量
- 专业程度

#### 3. 实体提取（5类）

自动提取问题中的关键实体：

| 类型 | 示例 |
|------|------|
| **公司** | 宁德时代、比亚迪、特斯拉 |
| **行业** | 新能源汽车、半导体、医药 |
| **产品** | iPhone、Model 3、比亚迪汉 |
| **概念** | 人工智能、元宇宙、碳中和 |
| **人物** | 马斯克、王传福、雷军 |

**技术实现：**
- 实体库匹配
- 上下文分析
- 歧义消除

## 🎯 Phase 2: 动态框架

### 核心功能

#### 1. 多复杂度框架（简单/中等/复杂）

为每种研究视角创建 3 种复杂度的框架：

**简单框架：**
- 核心要点
- 快速分析
- 适合事实性问题

**中等框架：**
- 完整分析
- 多维度评估
- 适合一般研究

**复杂框架：**
- 深度剖析
- 全面覆盖
- 适合专业研究

#### 2. 动态框架选择

根据问题复杂度自动选择合适的框架：

```
问题 → 复杂度评估 → 选择对应框架
         ↓
    简单 → 简单框架
    中等 → 中等框架
    复杂 → 复杂框架
```

**选择逻辑：**
- 简单问题 → 简单框架（快速回答）
- 中等问题 → 中等框架（平衡质量和速度）
- 复杂问题 → 复杂框架（深度研究）

#### 3. 实体到框架定制

根据提取的实体定制研究框架：

**示例：**
问题："分析宁德时代的竞争优势"
→ 提取实体：宁德时代（公司）
→ 定制框架：重点关注公司层面分析

**定制内容：**
- 研究关注点
- 分析维度
- 数据来源
- 输出重点

## 👤 Phase 3: 个性化

### 核心功能

#### 1. 用户历史分析

分析用户的历史使用记录：

**分析内容：**
- 常用研究视角
- 偏好问题类型
- 使用频率统计
- 研究时长分布

**数据来源：**
- 会话历史记录
- 用户选择偏好
- 反馈数据

**偏好置信度：**
- 高置信度（≥3次）→ 优先使用
- 中置信度（1-2次）→ 参考使用
- 低置信度（0次）→ 使用默认

#### 2. 个性化框架调整

根据用户偏好调整研究框架：

**调整策略：**
- 保持原框架为主
- 用户偏好为辅
- 渐进式调整
- 可追溯变更

**调整内容：**
- 研究重点
- 分析维度
- 输出格式
- 详细程度

#### 3. 用户偏好学习

自动学习用户的使用偏好：

**学习机制：**
- 记录每次使用的模式
- 检测连续3次相同模式
- 自动更新推荐模式
- 原子写入确保数据安全

**学习内容：**
- 偏好的研究视角
- 喜欢的问题类型
- 习惯的研究深度
- 常用的输出格式

## 🧠 Phase 4: 智能路由

### 核心功能

#### 1. 完整意图分类器（8维度）

从 8 个维度分析用户意图：

| 维度 | 说明 | 选项 |
|------|------|------|
| **问题类型** | 是什么/为什么/怎么办/对比/预测 | 5种 |
| **研究领域** | 金融/科技/消费/医疗/能源/工业/宏观 | 7种 |
| **时间跨度** | 短期/中期/长期 | 3种 |
| **复杂程度** | 简单/中等/复杂 | 3级 |
| **期望输出** | 总结/分析/建议/对比/预测/数据 | 6种 |
| **关键实体** | 公司/行业/产品/概念/人物 | 5类 |
| **紧急程度** | 正常/紧急/非常紧急 | 3级 |
| **技术水平** | 入门/中级/专业 | 3级 |

**新增检测函数：**
- `detectDomain()` - 研究领域检测
- `detectTimeHorizon()` - 时间跨度检测
- `detectOutputType()` - 期望输出类型检测
- `detectUrgency()` - 紧急程度检测
- `detectTechnicalLevel()` - 技术水平检测

#### 2. 多阶段路由决策

智能决策流程：

```
阶段1: 问题类型优先
    ↓ 根据问题类型选择基础模式

阶段2: 研究领域调整
    ↓ 根据研究领域优化模式选择

阶段3: 用户偏好融合
    ↓ 高置信度时优先用户偏好

阶段4: 紧急程度响应
    ↓ 紧急请求自动降低复杂度

阶段5: 技术水平适配
    ↓ 根据技术水平调整框架复杂度

最终决策: 最佳研究视角
```

**决策函数：**
- `makeRoutingDecision()` - 智能路由决策核心

#### 3. 智能提示词构建器

完整的提示词构建：

**特性：**
- 集成完整意图分析
- 自动智能路由决策
- 用户个性化适配
- 自动记录会话历史
- 完整的【智能分析】部分展示

**智能分析示例：**
```
【智能分析】
- 问题类型：建议型（怎么办）
- 研究领域：金融投资
- 时间跨度：中期（1-3个月）
- 复杂程度：中等
- 期望输出：深度分析
- 紧急程度：🟢 正常
- 技术水平：中级
- 路由决策：trader（建议型问题 → 理财顾问视角 → ...）
```

**构建函数：**
- `buildPromptSmart()` - 智能路由的完整提示词构建器
